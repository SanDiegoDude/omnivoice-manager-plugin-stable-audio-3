Stable Audio 3 — gated model troubleshooting

SA3 Medium weights are gated on Hugging Face. OmniVoice auto-downloads them the first time you generate (just like the built-in TTS models), but that download only works once you've accepted the license and your machine is authenticated. If you're seeing the "weights are gated" note in Sound Lab, one of those two pieces is missing.

1. Accept the license

  1. Open the model page: huggingface.co/stabilityai/stable-audio-3-medium
  2. Sign in to Hugging Face (top right).
  3. Read and accept the license / "Agree and access repository" prompt near the top of the page.
Access is usually granted instantly, but some gated repos require a short manual review. If the page says "Your request to access this repo has been submitted / is awaiting review", you'll have to wait for the approval email before the download can succeed.

2. Authenticate this machine (the most common cause)

Accepting the license in your browser is not enough — the download runs from the command line / sidecar, so it needs a token. Create a read token at huggingface.co/settings/tokens, then either log in once: 

huggingface-cli login
# paste your hf_... token when prompted

…or export it in the shell that launches OmniVoice before starting it:

export HF_TOKEN=hf_xxxxxxxxxxxxxxxxxxxxxxxx
# then (re)start the OmniVoice manager so the sidecar inherits it
If you already "signed in" in the browser but still get the gated error, this step is almost certainly what's missing — the CLI / sidecar has no token.

3. Confirm it worked

huggingface-cli whoami        # should print your username
huggingface-cli download stabilityai/stable-audio-3-medium --revision main \
  --include "model_config.json"   # should download, not 401/403

4. Manual / offline download (worst case)

If auto-download still won't cooperate, pull the weights yourself. The plug-in ships a helper that builds its env and fetches the model:

cd plugins/stable-audio-3
export HF_TOKEN=hf_xxxxxxxxxxxx
./bootstrap.sh --with-model

Or grab the snapshot directly with the HF CLI:

export HF_TOKEN=hf_xxxxxxxxxxxx
huggingface-cli download stabilityai/stable-audio-3-medium

Files land in your Hugging Face cache — $HF_HOME/hub or ~/.cache/huggingface/hub — under models--stabilityai--stable-audio-3-medium. Once a real *.safetensors weight file is present there, OmniVoice detects it and the gated note disappears on its own (no restart needed — it re-checks on the next Sound Lab open).

Common gotchas

  • Token has no read scope — recreate it as a Read token.
  • Behind a proxy / no internet on the box running OmniVoice — use the manual download on a connected machine and copy the cache folder over.
  • Disk full / interrupted download — look for stray *.incomplete files in the cache and delete them, then retry.
  • Wrong account — the account that accepted the license must match the token (huggingface-cli whoami).

Still stuck? Check the manager log for the exact Hugging Face error (GatedRepoError, 401, 403) — it tells you whether it's a license issue (accept it) or an auth issue (token).
